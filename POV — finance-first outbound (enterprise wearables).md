## 1. Когда использовать finance-first POV

| Сигнал аккаунта | Почему не «только ML» |
|-----------------|------------------------|
| Публичная компания, micro/small cap | IR narrative: **unit economics**, going concern, OEM win-rate |
| Убытки, OpEx cuts, restructuring | Покупатель ≠ только VP Engineering — нужны **CFO / CEO / BD** |
| AI в CES/IR, но выручка от hardware | AI = **variable COGS** и **R&D drag**, не feature slide |
| **S7** (Snapdragon on-glass) | On-device SDK TheStage ❌ — честный вход через **cloud optimize** (S1) |
| OEM / Quanta / Garmin-class partnerships | Продаём **OEM appendix с метриками**, не «заменим Qualcomm» |

**Не использовать как lead:** здоровые Series B voice apps без давления margin (там hybrid **W1** — MAU + latency, не going concern).

## 2. One-line POV (переиспользуемый)

> **TheStage helps wearable OEMs ship credible on-glass AI (voice, vision, translation) without linear growth in cloud inference COGS** — by optimizing models on **NVIDIA** (cloud / edge) today, and on **Apple companion / Jetson** only where architecture allows — so AI becomes a **margin story**, not only a demo story.

**Сужение для S7/OEM:**

> **We don’t replace Qualcomm on the glasses; we make the cloud and companion paths you already depend on cheap enough to put AI in every reference design.**

---

## 3. Larger narrative (для CEO / CFO / IR)

| Их макро-язык | Ваш thread |
|---------------|------------|
| Going concern, OpEx cuts | AI must **lower burn per shipped unit**, not add a cost center |
| Differentiation vs Meta / Apple | **Usable AI** with published **$ / 1k voice sessions** and latency SLO |
| OEM / strategic partnerships | **Benchmark pack** для RFP — AI as spec, not slideware |
| Software & services ecosystem | TheStage = **inference layer** (Elastic + ANNA + deploy), not another app store |
| Supply chain / inventory | Лёгкий акцент: fewer deployment variants (secondary) |

**Spine:** *Profitable AI for wearable OEMs = optimized inference economics, not more features.*

---

## 4. Акценты vs анти-акценты

### Делать акцент

1. **Variable cost** — $/inference, $/1k voice minutes, GPU-hours per OEM pilot  
2. **OEM enablement** — win design with **measurable AI SLOs**  
3. **Time-to-SKU** — fewer ML engineers per OEM variant (R&D efficiency, not «cut payroll»)  
4. **Voice + wearables** — aligns with TheStage founder narrative  
5. **Phased PoC** — one cloud model, 2–3 weeks, investor-mentionable directional metrics  
6. **Architecture honesty** — Snapdragon on-glass acknowledged; wedge = cloud (+ phone path if verified)

### Не вести разговор

- «Сократим ваш $8M OpEx» (неправдоподобно)  
- Full on-device orchestration **as shipped today**  
- Замена Snapdragon / их voice stack  
- Platform GPU rental as **lead** (sounds like more OpEx)  
- Consumer Brilliant / iPhone-hub story as primary archetype  
- Big-bang migration

## 6. Research & prep checklist (до первого касания)

### 6.1 Finance & IR (не делегировать только технарям)

- [ ] Последние **10-K / 10-Q**: revenue, gross margin, OpEx, cash runway, going-concern language  
- [ ] **Earnings call** summary: analyst questions (AI, partnerships, cost cuts)  
- [ ] **Stock pressure**: YTD, guidance cuts, restructuring announcements  
- [ ] **Revenue mix**: hardware vs software/services vs licensing (waveguide IP)  
- [ ] **Partnership bets** named on calls (e.g. Quanta, Garmin) — what AI means for those SKUs  
- [ ] **Impairments / inventory** — avoid claiming supply-chain fix; note only if relevant to R&D waste  

### 6.2 Product & technographic (для credibility)

- [ ] SKU map: which products have **voice / translate / vision AI** (CES, product pages)  
- [ ] **Where inference runs**: cloud vs on-device SoC (e.g. Snapdragon AR1) vs phone companion  
- [ ] Cloud provider signals (AWS, Nebius, Azure) — job posts, engineering blogs  
- [ ] Assign **S-segment** ([brief §3.11](Stage%20AI%20%E2%80%94%20understanding%20brief.md#311-icp-четыре-уровня-s--p--l--w))  
- [ ] **Buyer map**: CEO, CFO, VP BD/OEM, VP Eng, ML lead (if any)  

### 6.3 TheStage internal prep

- [ ] **One layer** for first meeting: **A (Optimize)** on NVIDIA cloud  
- [ ] Pick **one model** for PoC story (e.g. Whisper tier, small LLM) — match their demo stack  
- [ ] **Prod vs roadmap** checklist signed ([Roadmap](StageAI%20Roadmap.md) неделя 2)  
- [ ] **Pricing narrative** one slide: not $20/mo SaaS lead — per-GPU / PoC / enterprise  
- [ ] Legal: what case studies can be named (Brilliant, Praktika, etc.)  

### 6.4 Messaging matrix (finance row)

| Persona | Lead metric | Lead fear | CTA |
|---------|-------------|-----------|-----|
| **CFO** | $/1k inference sessions; margin per OEM pilot | Variable COGS of AI pilots | 30 min + benchmark scope |
| **CEO** | OEM win-rate; AI in partnership decks | «AI without P&L» | Executive brief 1-pager |
| **VP BD / OEM** | Latency + $ spec sheet for RFP | Losing design to Meta/XR | Joint benchmark for reference design |
| **VP Eng / ML** | tps, ttft, tier S/M/L | Export hell, duplicate stacks | Technical PoC scope doc |

## 7. Discovery questions (first call)

1. Where does **inference run today** for voice/translate — cloud, on-glass SoC, phone?  
2. Order of magnitude: **monthly cloud inference spend** vs optics R&D?  
3. Is AI **revenue** (license/services) or **cost center** to win hardware?  
4. Which **SKU** is the partnership bet (12 months)?  
5. Who owns ML — internal, ODM, third-party ASR/LLM?  
6. What metric would **CFO accept** on a pilot success memo? ($/session, p95 latency, both?)

## 9. Worked example — Vuzix (VUZI)

**Полная версия (MDP + POV + business case + unit economics):** [Vuzix — POV and business case (illustration)](Vuzix%20%E2%80%94%20POV%20and%20business%20case%20(illustration).md) — Parts 0–6, metrics registry, `12k × $5` scenarios, 100k footnote, mock exec flow.

**Sources:** Executive research summary (May 2026); public CES/IR; [understanding brief §3.10](Stage%20AI%20%E2%80%94%20understanding%20brief.md#310-prospect-map-кому-питчить-on-device-vs-cloud-не-клиенты).

### Customer snapshot

- AR / smart glasses OEM; waveguide + microLED; **~100–150 FTE**; NASDAQ microcap  
- **S7 → W3**; Ultralite / LX1 with **Snapdragon AR1**, voice & translation in positioning  
- Financial stress: large net losses, going-concern narrative, OpEx reduction (~$8M target), payroll plans  
- Strategy: OEM reference designs, partnerships (Quanta, Garmin mentioned), verticals (healthcare, security, defense)

### POV (finance-first)

**Situation:** Vuzix sells **partnership and optics** while the street asks when **AI revenue beats AI burn**.

**Complication:** CES AI story (voice on AR1) risks **cloud COGS per OEM pilot** without a published unit-economic spec — pilots don’t convert to licensed SKUs.

**Insight:** Reference-design winners will publish **AI SLOs + $/1k sessions** like FOV and battery — inference must be engineered, not demo-wrapped.

**Recommendation:**

| Phase | Action | Owner |
|-------|--------|-------|
| **0** (2–3 wk) | Baseline $ + p95 on **one** cloud voice model they use in demos | Eng + Voitech |
| **1** | Elastic/ANNA path; OEM benchmark sheet | Eng |
| **2** | Companion iOS or Jetson **only** if SKU architecture confirms | Product |

**Do not promise:** Snapdragon on-glass compile; OpEx reduction of $8M; replacing Qualcomm stack.

### Sample outreach (email)

> Vuzix is positioning Ultralite and LX1 around **voice AI and translation** while investors focus on the balance sheet. We often see OEM programs where **cloud inference cost per device** isn’t modeled before AI becomes a standard reference-design feature — so pilots don’t scale to licensed SKUs.  
>  
> TheStage optimizes **voice and multimodal inference on NVIDIA** (Elastic + ANNA) so you can publish **latency and $/1k sessions** OEMs can underwrite — without claiming to replace your on-glass Snapdragon stack.  
>  
> Worth 30 minutes to benchmark **one** model you’re already running in cloud for OEM demos?

### Multi-threading

| Thread | Hook |
|--------|------|
| **CFO / finance** | Variable COGS of AI pilots; directional $/1k sessions |
| **CEO** | AI that survives earnings narrative — metric, not demo |
| **BD / OEM** | Appendix for Quanta/Garmin-class designs |
| **Engineering** | Tier S/M/L, 2–3 week PoC — **after** exec sponsor |

## 11. Vuzix: где наибольшие benefits & cost savings

TheStage **не** сокращает $8M OpEx, payroll или BOM очков. Savings — **variable inference** и **OEM economics**.

### 11.1 Карта стека vs fit

| Слой | Пример (Vuzix) | TheStage fit | Savings potential |
|------|----------------|--------------|-------------------|
| M400 industrial | Embedded Speech SDK (local Android) | ❌ | ~0 |
| Ultralite / LX1 | AI on **Snapdragon AR1** on-glass | ❌ self-serve | ~0 today |
| Partner cloud AI (e.g. Ramblr) | Video + contextual guidance on GPU | ✅ **S1** | **High** |
| OEM reference designs | AI in Quanta/Garmin-class decks | ✅ **S1** + spec | **High** (strategic) |
| Internal R&D / CES demos | GPU cloud benchmarks | ✅ Elastic/ANNA | Medium |
| iPhone companion | Not confirmed primary | ⚠️ S4 | Medium if exists |
| Defense / Jetson edge | Vertical from IR | ⚠️ S5 | Niche |

### 11.2 Топ зон savings (приоритет)

| # | Зона | Формула / логика | Продукт TheStage |
|---|------|------------------|------------------|
| **1** | **Cloud inference** (partners, enterprise backends) | `cloud_spend × (1 - 1/N)` при N× throughput (PoC) | Elastic + ANNA + Qlip |
| **2** | **OEM pilots** | Меньше cloud **subsidy** per trial; faster SKU | Benchmark + tiers |
| **3** | **R&D GPU** | Fewer GPU-hours per iteration | Platform + Elastic |

### 11.3 Где savings ≈ 0 (говорить честно)

- On-glass **Qualcomm / AR1** compile  
- Legacy **embedded wake-word** stack на industrial SKU  
- Waveguide manufacturing, inventory, headcount restructuring  

### 11.4 Discovery (обязательно до цифры)

1. Monthly **cloud/GPU** spend on AI?  
2. Ultralite: inference **only on-glass** or **backend** too?  
3. OEM pilot: who pays inference — Vuzix, OEM, end customer?  
4. Most expensive workload: STT / LLM / vision / translation?  

**Deprioritize account** if: all new AI is on-device only, zero cloud bill, no near-term cloud roadmap.

## 12. Executive vs ML: двухуровневая продажа

### 12.1 Когда планировать CEO / CFO

| Сигнал | Exec involvement |
|--------|----------------|
| Public micro/small cap + going-concern language | **Plan for** CEO/CFO sponsor |
| AI prominent in IR, hardware revenue | CEO (story) + CFO (COGS) |
| OEM licensing / partnerships | + VP BD |
| Small PoC, eng culture | Often **eng-led** only |

**Vs W1 (Praktika-like):** ML/product first; CFO joins at scale — different playbook.

### 12.2 Value by persona (один PoC, два deliverable)

| Персона | Lead | Artifact |
|---------|------|----------|
| **CEO** | Strategic AI metric; partnership credibility | 1-pager for IR/partners |
| **CFO** | $/1k sessions; pilot ROI | Finance memo |
| **VP BD/OEM** | RFP-grade AI spec | Benchmark sheet |
| **ML / platform** | Performance; deploy time | Tech report (tiers, tps, ttft) |

### 12.3 Потоки (3-thread model)

```
[1] Executive — CEO/CFO/BD
    Hook: P&L AI vs demo AI · OEM unit economics
    Out: 20 min · Phase 0 scope ($)

[2] Technical — VP Eng / ML
    Hook: 1 model · 2–3 week benchmark
    Out: tech report · tier recommendation

[3] Sponsor sync
    Exec sees finance slide · Eng confirms feasibility → Phase 1
```

### 12.4 Ошибки

| Don't | Do |
|-------|-----|
| CEO email leads with tier S/M/L | Exec = $/session; tiers in appendix |
| Bypass ML after exec yes | CEO intro → VP Eng same week |
| Promise CFO «−$8M OpEx» | Variable inference + OEM pilot COGS only |
| Different promises to exec vs eng | Single PoC scope doc |

### 12.5 CRM

Tag deals: `buyer_motion: exec-led | eng-led | hybrid`

Ask TheStage on kickoff: **«На OEM deals economic buyer — CEO, CFO, or VP Eng?»**
