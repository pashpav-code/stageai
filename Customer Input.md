## Модель монетизации

- **Usage-based** — стандарт для AI/infra-сервисов.
- Оплата завязана на **число задеплоенных GPU**, которые используют софт заказчика.
- Пример: **~$1 000/год за GPU** за софт (сама GPU — отдельно, у провайдера).
- Типичный диапазон по on-prem/деплою: **~30–100 GPU/год**.
- Облако (Nebius и аналоги) — **on-demand**, объёмы ожидают **существенно выше**.

## Спрос: voice / on-device

- Сильный повторяющийся паттерн: запросы на **voice-ассистента** и смежное:

- Типичные сценарии: **мобильные приложения**, **AI tutor**, **wearables** (умные очки, кольца и т.д.).

## Позиционирование продукта

- **Оркестрация + SDK** для model providers, которые хотят **on-device** на смартфоне.
- Ценность: **оптимизация/сжатие моделей** под скорость + **marketplace** оптимизированных моделей.
- **Inference engine + orchestration**: связка моделей в цепочку (пример voice: STT → LLM → TTS).
- Клиент **собирает конфиг** из выбранных моделей (условно AssemblyAI на транскрипцию, LLM Labs on-device и т.д.).
- **Day-zero support** для партнёров; партнёрства с сильными model teams.
- Аналог в облаке: **LiveKit** — они строят похожее, но **on-device**.
